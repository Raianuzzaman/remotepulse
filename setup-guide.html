Step 01
Create a Free GitHub Account & Repository
GitHub hosts your code AND runs it for free every day via GitHub Actions.
✓
  • Go to github.com → Click Sign Up → Create free account
  • After login, click the + icon (top right) → New repository
  • Name it: remotepulse
  • Set to Public (required for free GitHub Pages hosting)
  • Check "Add a README file" → Click Create repository
💡 Why GitHub? GitHub gives you 2,000 free minutes/month to run automated scripts. Your daily job fetch uses about 5 minutes/day = 150 min/month. Well within the free limit.
🗄️
Step 02
Set Up Supabase Database
Free PostgreSQL database to store all your jobs. 500MB free — you'll never fill it.
✓
  • Go to supabase.com → Click Start your project → Sign up free
  • Click New Project → Choose any organization → Name it remotepulse
  • Set a database password (save it somewhere!) → Choose region closest to you → Click Create new project
  • Wait ~2 minutes for it to set up
  • Once ready, click SQL Editor in the left sidebar
  • Click + New query → paste the SQL below → click Run
SQL — Run in Supabase SQL Editor
-- Create jobs table
CREATE TABLE IF NOT EXISTS jobs (
  id          BIGSERIAL PRIMARY KEY,
  title       TEXT NOT NULL,
  company     TEXT,
  location    TEXT DEFAULT 'Worldwide',
  site        TEXT,
  salary      TEXT,
  job_type    TEXT,
  url         TEXT UNIQUE NOT NULL,
  date_posted DATE,
  description TEXT,
  fetched_at  TIMESTAMPTZ DEFAULT NOW()
);

-- Create applied jobs tracker
CREATE TABLE IF NOT EXISTS applied_jobs (
  id         BIGSERIAL PRIMARY KEY,
  job_url    TEXT UNIQUE NOT NULL,
  job_title  TEXT,
  company    TEXT,
  applied_at TIMESTAMPTZ DEFAULT NOW(),
  status     TEXT DEFAULT 'Applied'
);

-- Allow public access (needed for dashboard)
ALTER TABLE jobs         ENABLE ROW LEVEL SECURITY;
ALTER TABLE applied_jobs  ENABLE ROW LEVEL SECURITY;

CREATE POLICY "read jobs"   ON jobs         FOR SELECT USING (true);
CREATE POLICY "insert jobs"  ON jobs         FOR INSERT WITH CHECK (true);
CREATE POLICY "update jobs"  ON jobs         FOR UPDATE USING (true);
CREATE POLICY "all applied"  ON applied_jobs FOR ALL    USING (true);

-- Verify it worked
SELECT 'Tables created!' as status;

Step 03
Create Your Telegram Bot (Get Notified on Phone)
Every morning you'll get a Telegram message with new job listings. Takes 3 minutes.
✓
  • Open Telegram on your phone → Search for @BotFather → Start chat
  • Send the message: /newbot
  • BotFather asks for a name → type anything e.g. My Job Alerts
  • BotFather asks for a username → type e.g. myjobpulse_bot (must end in _bot)
  • BotFather sends you a token like: 7123456789:AAGxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx → save this!
  • Now search for your new bot → open it → click Start
  • To get your Chat ID, open this URL in browser (replace TOKEN with yours):
URL — Open in browser
https://api.telegram.org/botYOUR_TOKEN_HERE/getUpdates
  • You'll see JSON — find "id" inside "chat" — that number is your Chat ID
  • If the JSON is empty, send any message to your bot first, then reload the URL
  • Save both: Bot Token and Chat ID
⚠️ Keep your Bot Token private! Anyone with it can send messages through your bot. Never share it publicly.
🔐
Step 04
Upload Your Files & Add GitHub Secrets
Add your credentials as encrypted secrets — GitHub keeps them safe and passes them to your script.
✓
Part A — Upload your project files
  • In your GitHub repo, click Add file → Upload files
  • Upload all files from the RemotePulse-Cloud.zip (maintaining folder structure)
  • Or use GitHub Desktop app for easier drag-and-drop uploading
  • Commit the files with message: "Initial commit"
Part B — Add secrets (your credentials)
  • In your repo, click Settings (top menu)
  • Left sidebar → Secrets and variables → Actions
  • Click New repository secret — add each of these 4 secrets:
Secret NameValue (where to find it)
${[ ['SUPABASE_URL','Your Supabase Project URL (Step 2)'], ['SUPABASE_ANON_KEY','Your Supabase anon public key (Step 2)'], ['TELEGRAM_BOT_TOKEN','Your bot token from BotFather (Step 3)'], ['TELEGRAM_CHAT_ID','Your chat ID number (Step 3)'], ].map(([name,val])=>`
${name} ${val}
`).join('')}
✅ Secrets are encrypted. GitHub never shows them again after saving — not even to you. Your credentials are completely safe.
🌐
Step 05
Host Your Dashboard on GitHub Pages (Free Website)
Your dashboard becomes a real website accessible from anywhere — phone, laptop, any browser.
✓
  • In your GitHub repo, click Settings
  • Scroll down the left sidebar → click Pages
  • Under Source → select Deploy from a branch
  • Under Branch → select main → folder: /dashboard → Click Save
  • Wait 2–3 minutes → GitHub shows you a URL like:
🌐 https://your-username.github.io/remotepulse/
  • Open that URL → go to the "Connect Supabase" tab in the dashboard
  • Enter your Supabase URL and anon key → click Save & Connect
  • Your dashboard is now live! Bookmark it on your phone.
Also save your Dashboard URL as a GitHub Secret
  • Go back to Settings → Secrets → New secret
  • Name: DASHBOARD_URL → Value: your GitHub Pages URL
  • This lets Telegram messages link directly to your dashboard
🚀
Step 06
Trigger Your First Job Fetch
Test everything works before waiting for tomorrow's automatic run.
✓
  • In your GitHub repo, click the Actions tab (top menu)
  • You'll see "RemotePulse — Daily Job Fetch" in the list
  • Click it → click "Run workflow" button (top right) → Run workflow
  • A yellow dot appears — click it to watch the live logs
  • It takes about 3–8 minutes to scrape all job boards
  • When it turns ✅ green — check your Telegram for a notification!
  • Open your dashboard website — jobs should now be loading
⚠️ If it fails (red ✗): Click the failed run → read the error message. Most common issue is a typo in a secret name. Double-check Step 4.
⏰ After this, it runs automatically every day at 9:00 AM Bangladesh time (3:00 AM UTC) — no action needed from you ever again.
